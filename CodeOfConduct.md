WM3J ARS Suite – Code of Conduct
1. Purpose
The WM3J ARS Suite supports amateur radio operators with reliable, operator‑centric tools for contesting, mapping, digital modes, and station integration. This Code of Conduct ensures that all contributors, users, and maintainers interact in a respectful, safe, and productive environment that reflects the values of the amateur radio community.

2. Expected Behavior
All participants in the ARS Suite ecosystem are expected to:

Act with professionalism and respect toward all contributors.

Provide constructive feedback focused on improving code, documentation, or user experience.

Collaborate openly and share knowledge.

Follow project standards, including coding style, documentation requirements, and module architecture.

Protect the integrity of the project by reporting bugs, vulnerabilities, or misuse responsibly.

Honor the spirit of amateur radio, including experimentation, learning, and mutual support.

3. Unacceptable Behavior
The following behaviors are not tolerated:

Harassment, discrimination, or personal attacks.

Dismissive or hostile responses to questions or contributions.

Knowingly submitting malicious code, backdoors, or harmful modules.

Misrepresenting ARS Suite as proprietary, paid, or restricted software.

Using ARS Suite tools for illegal, unethical, or non‑amateur‑radio purposes.

Spamming issues, pull requests, or discussions.

Circumventing licensing terms or removing required attribution.

4. Contribution Standards
To maintain a clean, modular, and operator‑centric ecosystem:

Write clear, maintainable code aligned with ARS Suite architecture.

Document modules, UI elements, and configuration options.

Submit complete pull requests with testing notes and rationale.

Respect modular boundaries; each J‑module should remain focused and self‑contained.

Avoid partial fixes; provide full‑file or full‑module regeneration when appropriate.

Preserve UI consistency, including cockpit‑style layouts and grid alignment.

5. Reporting Issues or Violations
If you experience or witness behavior that violates this Code of Conduct:

Submit a confidential report through GitHub’s private contact options, or

Contact the project maintainer directly using the email listed in the repository.

Reports should include:

A description of the incident

Links, screenshots, or logs if available

Names or usernames of involved parties

Any relevant context

All reports will be reviewed promptly and handled with discretion.

6. Enforcement
Depending on severity, maintainers may take the following actions:

Verbal or written warning

Request for retraction or correction

Temporary or permanent ban from contributing

Rejection or rollback of harmful contributions

Escalation to GitHub support for severe violations

Enforcement decisions are final to maintain project integrity.

7. Licensing and Ownership
The WM3J ARS Suite is:

Free to use

Open source

Licensed under GPLv3

Open for personal modification, extension, and development

All contributions are licensed under GPLv3 and may be modified, redistributed, or integrated into the ARS Suite ecosystem.

8. Commitment to the Amateur Radio Community
The ARS Suite is built by operators, for operators. The project is committed to:

Supporting experimentation and learning

Encouraging new contributors

Maintaining transparent development practices

Upholding the values of the amateur radio service
